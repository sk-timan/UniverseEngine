## 新增需求

### 需求:系统必须提供运行时反射元数据描述符
系统必须为每个已注册的反射类维护 `UClass` 元数据，包含类名、父类链、属性链表与函数链表；系统禁止仅依赖字符串类名比较来访问成员字段。

#### 场景:查询类属性列表
- **当** 调用方持有有效 `UClass` 且该类已通过反射注册
- **那么** 系统必须能遍历其全部 `FProperty` 描述符，并返回属性名、类型、标志位与内存偏移

#### 场景:按名称查找属性
- **当** 调用方对某 `UClass` 调用 `FindPropertyByName` 且名称存在
- **那么** 系统必须返回对应 `FProperty`；名称不存在时必须返回空且不崩溃

### 需求:系统必须支持基于偏移的属性读写
系统必须通过 `FProperty` 描述符在任意 `UObject` 实例上读写标记成员，禁止要求调用方手写 `static_cast` 到具体子类访问字段。

#### 场景:读取浮点属性
- **当** 属性类型为 `float` 或 `double` 且实例指针有效
- **那么** 系统必须返回该字段当前数值，且与直接 C++ 成员访问结果一致

#### 场景:写入可编辑属性
- **当** 调用方通过 `FProperty::SetValue` 写入与属性类型匹配的值
- **那么** 系统必须更新实例内存，且后续读取得到新值

#### 场景:类型不匹配时拒绝写入
- **当** 调用方尝试以不兼容类型写入属性
- **那么** 系统必须拒绝操作并返回可诊断错误，禁止静默内存破坏

### 需求:系统必须提供反射类型注册表
系统必须维护全局 `FReflectionRegistry`，支持按类名字符串查找 `UClass`；所有生成或手工注册的反射类必须在模块初始化阶段完成注册。

#### 场景:StaticClass 返回稳定单例
- **当** 多次调用某类的 `StaticClass()`
- **那么** 系统必须返回同一 `UClass` 实例，且其父类链与编译期继承关系一致

#### 场景:IsA 与反射父类一致
- **当** 对象属于某反射子类
- **那么** `IsA(ParentClass)` 必须沿 `UClass` 父类链正确判定，与现有 `UClass::IsChildOf` 行为兼容

### 需求:系统必须支持枚举与结构体反射描述
系统必须为 `UENUM` 标记的枚举提供名称↔数值映射；必须为 `USTRUCT` 标记的结构体（如 `FVector`）提供 `UScriptStruct` 描述以支持嵌套属性序列化与编辑器展示。

#### 场景:枚举按名称解析
- **当** 编辑器或序列化传入合法枚举项名称
- **那么** 系统必须解析为对应整型值；非法名称必须失败并报告

#### 场景:结构体属性按子字段访问
- **当** 属性类型为已注册的 `UScriptStruct`
- **那么** 系统必须支持按子字段偏移读写（至少支持 `FVector` 的 X/Y/Z）
