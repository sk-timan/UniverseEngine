## 目的

在应用窗口顶部添加工具栏菜单，提供新建地图、打开地图、保存地图等交互入口，完善应用的地图管理功能。

## 需求

### 需求：MainWindow 必须提供菜单栏

MainWindow 必须使用 Qt QMenuBar 组件在窗口顶部提供应用菜单。

#### 场景：菜单栏结构

- **当** MainWindow 初始化
- **那么** 必须创建菜单栏，包含"文件"菜单

### 需求："文件"菜单必须包含标准操作

"文件"菜单必须包含：新建地图、打开地图、保存地图、另存为、退出。

#### 场景：菜单项列表

- **当** 用户点击"文件"菜单
- **那么** 下拉菜单必须显示：
  - 新建地图
  - 打开地图
  - 分隔线
  - 保存地图
  - 另存为
  - 分隔线
  - 退出

#### 场景：菜单快捷键

- **当** 显示菜单项
- **那么** 必须显示对应的快捷键：
  - 新建地图：Ctrl+N
  - 打开地图：Ctrl+O
  - 保存地图：Ctrl+S
  - 另存为：Ctrl+Shift+S

### 需求：菜单操作必须调用正确的应用接口

每个菜单项必须连接到对应的 GameApp 接口，实现地图管理功能。

#### 场景：新建地图

- **当** 用户点击"新建地图"
- **那么** 必须调用 GameApp::CreateNewLevel() 创建新关卡

#### 场景：打开地图

- **当** 用户点击"打开地图"
- **那么** 必须弹出文件选择对话框，选择 JSON 文件后调用 GameApp::LoadLevelFromFile()

#### 场景：保存地图

- **当** 用户点击"保存地图"
- **那么** 如果当前关卡有对应文件则直接保存，否则弹出另存为对话框

#### 场景：另存为

- **当** 用户点击"另存为"
- **那么** 必须弹出文件保存对话框，选择路径后保存当前关卡数据

#### 场景：退出

- **当** 用户点击"退出"
- **那么** 必须触发应用关闭流程

### 需求：文件对话框必须限制为 JSON 格式

系统必须使用 QFileDialog 限制用户只能选择或保存 JSON 文件。

#### 场景：打开文件对话框过滤

- **当** 弹出打开文件对话框
- **那么** 必须设置文件过滤器为 "Map Files (*.json)"

#### 场景：保存文件对话框过滤

- **当** 弹出保存文件对话框
- **那么** 必须设置默认扩展名为 ".json"
